# NW Iowa + SE South Dakota Wildfire Risk Prototype

This prototype has three tabs: Home (historical fires), Model (risk surface), and Expert Elicitation (paused placeholder while model tuning is in progress).

## What is in the app

- Historical fire context points (`docs/data/historical_fires.geojson`)
- Chunk index (`docs/data/risk_chunks/index.geojson`)
- Chunked 1 km risk cells (`docs/data/risk_chunks/chunks/*.geojson`)
- Satellite NDVI overlay from NASA GIBS (dated `2026-03-28`)

## Generate risk grid

Run:

```bash
python3 scripts/generate_risk_grid.py
```

The script generates approximately 1 km square cells and writes chunked GeoJSON for viewport-based loading.  
It uses real historical weather from Open-Meteo Archive API for each weather sample point and fails if data cannot be fetched.

Optional offline testing mode:

```bash
python3 scripts/generate_risk_grid.py --allow-fallback
```

Data mode values:

- `data_mode = "open_meteo_archive"` in strict real-data runs
- `data_mode = "offline_fallback"` only when `--allow-fallback` is explicitly used

## Historical Fire Ingestion (Real Data Only)

Default ingestion area covers NW Iowa + SE South Dakota (`bbox: -97.60,41.80,-94.90,43.85`).

Source #1 (perimeter centroids from authoritative historical perimeters):

```bash
python3 scripts/ingest_historical_fires.py --start-year 2004 --end-year 2023
```

Source #2 (NASA FIRMS active detections) requires a MAP_KEY from FIRMS:

```bash
python3 scripts/ingest_historical_fires.py \
  --start-year 2004 --end-year 2023 \
  --include-firms \
  --firms-map-key YOUR_FIRMS_MAP_KEY
```

Notes:

- The script writes only sourced records with provenance fields.
- No synthetic rows are generated by the ingestion script.

## Historical Fire Analysis (Model Calibration)

Use this to test whether the index structure separates known fires from nearby non-fire controls.

Inputs:

- Fire template: `data/input/fires_template.csv`
- Optional NDVI template: `data/input/ndvi_template.csv`

Run:

```bash
python3 scripts/analyze_historical_fires.py \
  --fires-csv data/input/fires_template.csv \
  --ndvi-csv data/input/ndvi_template.csv \
  --output-dir analysis_output
```

Outputs:

- `analysis_output/historical_fire_features.csv` (event/control feature table)
- `analysis_output/summary.json` (AUC + score separation summary)

If you do not have NDVI history yet, omit `--ndvi-csv`.  
The script still evaluates weather + shoulder-season features so you can iterate immediately.

## Build Model Table (From Real Ingested Points)

Create a filtered, deduped training table from `docs/data/historical_fires.geojson`:

```bash
python3 scripts/build_model_table.py \
  --input-geojson docs/data/historical_fires.geojson \
  --output-dir analysis_output \
  --start-year 2004 --end-year 2023 \
  --season spring_fall \
  --control-mode hard_spatial \
  --hard-negative-min-km 18 \
  --join-weather \
  --weather-sample-km 8
```

Outputs:

- `analysis_output/model_table.csv`
- `analysis_output/model_table_meta.json`

Notes:

- FIRMS detections are spatiotemporally deduped into event clusters.
- Source provenance and source weights are retained.
- No synthetic incidents are created; synthetic controls are generated as training negatives.
- Default controls are **hard spatial negatives** (random location in the Iowa/SE SD bbox, at least `--hard-negative-min-km` from any positive).

## Train Baseline Model

Train a first baseline logistic model with a year-based holdout split:

```bash
python3 scripts/train_baseline_model.py \
  --input-csv analysis_output/model_table.csv \
  --output-dir analysis_output \
  --test-split year \
  --test-start-year 2021
```

Optional spatial holdout (longitude split):

```bash
python3 scripts/train_baseline_model.py \
  --input-csv analysis_output/model_table.csv \
  --test-split spatial \
  --spatial-test-min-lon -96.0
```

Outputs:

- `analysis_output/model_metrics.json`
- `analysis_output/model_coefficients.csv`
- `analysis_output/model_scored_test.csv`

## One-Pass RStudio Pipeline

If you prefer RStudio, run the full pipeline from a single script:

```r
source("run_pipeline.R")
```

What it does:

- Ingest historical fires (`docs/data/historical_fires.geojson`)
- Build model table (`analysis_output/model_table.csv`)
- Downsample model rows (full-area retained, fewer training plots)
- Train baseline model (`analysis_output/model_metrics.json`)
- Generate historical risk grid and next-week forecast chunks
- Derive run area from `data/demo_counties.shp` bbox (when available)

The script is organized as a single entrypoint so the same command can later be reused by GitHub Actions.
